## Install & Authenticate with Github CLI
Sponsor packages require an active sponsorship while public packages wont.

Since sponsor repositories are assigned to a user, sponsor packages will required your local development machine to be authenticated with Github CLI.

If you're not already authenticated with the Github CLI, follow the steps outlined in the [Github CLI getting started guide](https://docs.github.com/en/enterprise-cloud@latest/github-cli/github-cli/quickstart).
